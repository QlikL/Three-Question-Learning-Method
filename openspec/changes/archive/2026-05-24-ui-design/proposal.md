# UI Design Proposal - 三问高效学习机

## 背景
"三问高效学习机"是一款以提问触发课程生成、基于"三问认知引擎"帮助用户构建个性化学习路径的智能学习工具。产品需求书已明确定义了五大核心功能模块：柔性课程生成、复合知识库构建、三问认知引擎、学习进度追踪、用户交互。

当前项目处于**界面UI设计**阶段，需要将需求书中的功能描述转化为可视化的Web界面。

## 目标
设计并实现一套完整的前端UI界面，覆盖产品需求书中定义的所有功能模块，包括：
1. **主界面（初始状态）** - 搜索框与引导语
2. **学习空间界面** - 知识图谱、争议点对比、资料区
3. **测评中心界面** - 测评列表、答题界面、结果反馈
4. **个人中心界面** - 学习数据总览、课程归档、资料上传历史

## 范围

### In Scope
- 所有页面的HTML结构实现
- CSS样式设计（响应式布局、视觉主题）
- 页面间导航与路由逻辑
- 模拟数据驱动的组件展示
- 与localStorage/IndexedDB的数据交互接口

### Out of Scope
- 真实AI API接入（后续替换）
- 后端服务与数据库
- 用户认证系统
- 多端数据同步

## 设计原则
1. **简洁优先** - 初始状态仅有搜索框和引导语，避免信息过载
2. **渐进呈现** - 功能随用户操作逐步展示，未创建的课程不显示
3. **反馈即时** - 用户操作后即时更新UI状态（课程生成、进度变化）
4. **模块解耦** - 各功能模块独立实现，便于后续扩展和维护

## 技术方案
| 层面 | 选择 | 理由 |
|------|------|------|
| 样式 | 原生CSS + CSS Variables | 无框架依赖，易于维护主题变量 |
| 路由 | 原生Hash路由 | 纯前端SPA，无需后端支持 |
| 存储 | IndexedDB + localStorage | 结构化课程数据用IndexedDB，偏好设置用localStorage |
| 图表 | Canvas 自绘 | 避免第三方库依赖 |
| AI模拟 | JS模块+setTimeout | 模拟异步AI响应，后续替换为真实API调用 |

## 视觉风格建议
- 主色调：蓝色系（#2563EB）搭配浅灰背景
- 字体：系统默认字体栈
- 卡片设计：圆角12px，轻微阴影
- 响应式：适配桌面与平板

## 非目标（明确不做）
- 不做动画框架集成（仅用CSS过渡）
- 不做拖拽排序（长按操作用点击菜单替代）
- 不做实时协作
