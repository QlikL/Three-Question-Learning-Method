## Context

项目使用 IndexedDB 存储课程、知识图谱、测评、争议点、资料等数据，使用 localStorage 存储设置和主题。现有 `clear-cache.html` 只提供数据清除功能，无法浏览具体内容。开发调试时需要查看数据库中的实际数据。

## Goals / Non-Goals

**Goals:**
- 提供一个独立的 HTML 页面，可视化浏览 IndexedDB 中所有对象仓库的数据
- 支持查看 localStorage 中的设置数据
- 支持 JSON 格式化和原始数据切换
- 支持搜索过滤功能
- 页面风格与主应用一致（使用 CSS 变量）

**Non-Goals:**
- 不支持数据编辑/修改功能（只读查看）
- 不支持数据导入导出（已有其他入口）
- 不集成到主应用路由中（独立页面）

## Decisions

### 1. 独立 HTML 页面 vs 主应用内路由
**选择：独立 HTML 页面**
- 理由：不增加主应用代码体积，不暴露数据库查看入口给普通用户，开发调试时直接打开 `db-viewer.html` 即可
- 替代方案：在设置页面添加"查看数据库"入口 → 增加主应用复杂度，且调试工具不应暴露给用户

### 2. 数据展示方式
**选择：树形 JSON 折叠展示**
- 理由：JSON 数据天然适合树形展示，支持折叠/展开，易于阅读
- 替代方案：表格展示 → 不适合嵌套数据结构（如知识图谱）

### 3. 依赖方案
**选择：直接引用 `js/store.js` 复用 IndexedDB 连接逻辑**
- 理由：复用已有的 `Store.openDB()` 和对象仓库定义，避免重复代码
- 替代方案：独立实现 IndexedDB 连接 → 代码重复，维护成本高

## Risks / Trade-offs

- [大数据量性能] 大量课程数据可能导致页面渲染卡顿 → 每个对象仓库限制显示条数，支持分页加载
- [安全考虑] 数据库查看器可能暴露敏感信息（API Key 等）→ 仅在本地开发环境使用，不部署到生产环境
