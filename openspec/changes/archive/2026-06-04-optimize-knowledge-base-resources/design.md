## Context

当前知识库的AI补充资料功能通过 `AiService.generateMaterials()` 调用AI推荐5-8个学习资源，返回的 `items` 数组中混合了文字资料和视频资源，没有明确分类。`AiMock.mockGenerateMaterials()` 在此基础上还会额外生成2份AI总结文档。用户需要更丰富、更有组织的外部资源，包括至少5个文字资料链接（知乎、百度文库、菜鸟教程、GitHub等）和至少5个视频链接（B站、抖音等）。

## Goals / Non-Goals

**Goals:**
- AI提示词分别要求生成外部文字资料（至少5个）和学习视频（至少5个），各自独立返回
- 知识库页面AI补充资料区域按三个模块分区展示：AI总结文档、外部文字资料、学习视频
- 资料卡片根据类型显示不同的图标和来源标识
- 模拟数据（降级场景）也按新结构提供分类资源

**Non-Goals:**
- 不实现真实的网页爬取（前端无法跨域爬取），仍通过AI生成推荐链接
- 不修改用户上传资料的功能逻辑
- 不修改AI总结文档的生成逻辑（已实现）
- 不新增外部依赖或CDN引入

## Decisions

### 1. AI提示词拆分方案

**选择：** 将 `generateMaterials` 的提示词改为一次性要求AI分别返回 `textResources` 和 `videoResources` 两个数组

**理由：**
- 一次API调用即可获取两类资源，减少延迟和费用
- 提示词明确要求每类不少于5个，且指定具体来源平台
- 文字资料来源：知乎、百度文库、菜鸟教程、GitHub（计算机类）、CSDN、掘金等
- 视频来源：B站、抖音、慕课网、网易公开课、中国大学MOOC等

**替代方案：** 分两次API调用分别生成文字资料和视频 → 增加延迟和费用，不采用

### 2. 返回数据结构调整

**选择：** `generateMaterials` 返回的 `items` 数组中增加 `category` 字段区分资源类别

```javascript
{
  id: "mat_1",
  title: "资料标题",
  type: "ai",
  category: "text",  // 新增：'text' | 'video'
  format: "link",
  source: "知乎",
  url: "https://...",
  tags: [...]
}
```

**理由：**
- `category` 字段用于前端分组渲染，不影响现有 `type` 字段逻辑
- 保持与现有数据结构兼容

### 3. 页面分区渲染方案

**选择：** `knowledgeRepositoryPage` 中AI补充资料区域改为三个子区块，每个区块有独立标题，通过 `category` 字段过滤 `items` 数组

**渲染顺序：**
1. AI总结文档（`type === 'ai-summary'`）
2. 外部文字资料（`category === 'text'`）
3. 学习视频（`category === 'video'`）

**理由：**
- 复用现有 `materialList` 组件渲染逻辑，仅增加分组和标题
- AI总结文档已有独立类型标识，无需额外字段

## Risks / Trade-offs

- [AI生成的URL可能无效] → 这是已知限制，AI无法保证生成的URL真实可访问。提示词中明确要求"真实可访问的URL"作为最佳努力，用户点击后若链接失效可自行搜索替代。
- [资源数量不足5个] → 提示词要求"不少于5个"，但AI可能返回不足。在前端做兜底处理：若某类资源不足5个，仍正常展示已有资源，不阻塞页面。
- [分类字段缺失] → 对于历史已生成的资料数据（无 `category` 字段），前端根据 `format` 字段推断：`format === 'video'` 归为视频，其余归为文字资料。
