## Context

当前学习空间页面集成了四大部分：知识图谱、争议点、测评和复合知识库。随着功能不断完善，页面变得冗长，用户需要大量滚动才能访问所有模块。同时，复合知识库作为独立的学习资源中心，需要更多功能空间（如文件上传、分类管理等），但当前嵌套在学习页面中限制了其发展。

**当前状态：**
- 学习页面包含导航栏（三问）+ 学习区 + 资料区
- 资料区使用MaterialList组件展示AI生成和用户资料
- 点击资料卡片弹出模态框，显示详情和跳转链接
- 数据存储使用IndexedDB的materials表，以courseId为主键

**约束条件：**
- 保持现有资料数据模型兼容
- 使用原生JavaScript，不引入前端框架
- 保持模块化架构（components/pages/services分离）
- IndexedDB存储方案不变

## Goals / Non-Goals

**Goals：**
- 将复合知识库从学习页面解耦，成为独立页面
- 提供清晰的入口：课程卡片上的资料图标按钮
- 支持用户拖拽上传本地文件到知识库
- 保持现有的资料展示样式和交互体验
- 优化学习页面的空间利用率

**Non-Goals：**
- 不改变现有AI资料生成逻辑
- 不修改知识图谱、争议点、测评模块
- 不实现复杂的文件管理功能（如文件夹、版本控制）
- 不改变IndexedDB数据结构（仅扩展）

## Decisions

### 决策1：路由设计
**选择**：使用hash路由 `#/knowledge/:courseId`
**理由**：
- 与现有路由系统（#/home, #/learning/:courseId）保持一致
- 无需服务器配置，纯前端实现
- URL可直接分享和收藏

**替代方案**：
- 使用URL参数（#/learning/:courseId?tab=materials）- 会污染学习页面路由
- 使用模态框弹出 - 不适合大页面，影响URL管理

### 决策2：入口按钮位置
**选择**：在课程卡片的右上角添加资料图标按钮
**理由**：
- 不干扰课程卡片的原有功能（进入学习、删除）
- 视觉层级清晰，图标直观表达含义
- 与现有按钮风格统一

**替代方案**：
- 在课程卡片底部添加按钮 - 占用过多空间
- 在导航栏添加全局知识库入口 - 无法区分课程

### 决策3：文件上传实现
**选择**：使用HTML5 File API + FileReader，将文件内容Base64编码后存储到IndexedDB
**理由**：
- 纯前端实现，无需后端支持
- IndexedDB支持Blob存储，容量充足
- FileReader兼容性好，支持现代浏览器

**替代方案**：
- 使用FormData上传到服务器 - 项目无后端
- 仅存储文件引用（路径）- 浏览器安全限制无法访问本地文件

### 决策4：资料页面布局
**选择**：保持MaterialList组件的现有样式，仅新增上传区域
**理由**：
- 用户已熟悉当前交互
- 减少开发工作量
- 保持一致性

**新增内容**：
- 页面顶部添加"上传资料"区域（拖拽+点击选择）
- 上传进度提示
- 文件格式验证（pdf、doc、docx、txt、md）

## Risks / Trade-offs

### 风险1：大文件存储性能
**问题**：用户可能上传较大的PDF文件（>10MB），导致IndexedDB写入缓慢
**缓解**：
- 添加文件大小限制（建议<20MB）
- 显示上传进度
- 提供异步上传，不阻塞UI

### 风险2：路由冲突
**问题**：#/knowledge/:courseId可能与未来其他路由冲突
**缓解**：
- 在router.js中统一管理所有路由
- 使用参数校验（检查courseId是否存在）

### 风险3：旧课程卡片无资料按钮
**问题**：用户可能找不到如何访问知识库
**缓解**：
- 在课程卡片上添加明显的图标按钮
- 在首次使用时显示提示

### 权衡：文件存储 vs 文件引用
**选择存储文件内容**：
- ✅ 优点：离线可用、无需服务器、数据安全
- ❌ 缺点：占用浏览器存储空间、大文件性能差

如果未来需要支持超大文件，可以考虑引入服务器存储。

## Migration Plan

### 部署步骤
1. 创建knowledgeRepositoryPage.js和CSS文件
2. 修改router.js添加新路由
3. 修改courseCard.js添加资料按钮
4. 修改learningPage.js移除资料区
5. 扩展materialList.js添加上传功能
6. 测试所有场景（创建课程→添加资料→查看→上传→删除）

### 回滚策略
- 保留learningPage.js中的资料区代码（注释掉，不删除）
- 如遇问题，可快速恢复旧版
- 新路由不影响现有页面

### 数据迁移
- **无需迁移**：现有materials表数据直接可用
- 只需改变访问路径（从learningPage改为knowledgeRepositoryPage）

## Open Questions

1. **是否支持文件预览？**
   - 当前方案：点击后弹出模态框显示信息和跳转链接
   - 未来可能：PDF.js实现在线预览
   - 决策：MVP版本不支持，后续迭代

2. **是否支持批量上传？**
   - 当前方案：一次选择一个文件
   - 未来可能：多选文件或文件夹拖拽
   - 决策：MVP版本不支持，后续迭代

3. **是否限制用户上传资料数量？**
   - 当前方案：不限制
   - 风险：IndexedDB可能占满
   - 决策：暂不限制，监控使用情况，必要时添加提示
