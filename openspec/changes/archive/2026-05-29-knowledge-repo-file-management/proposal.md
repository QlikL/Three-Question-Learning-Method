# 知识库文件管理增强

## 背景

当前知识库页面已支持用户上传资料和查看AI生成资料，但存在三个功能缺失：

1. **用户上传的资料无法删除**：资料详情弹窗中只有"下载"和"关闭"按钮，没有删除功能。用户误传或需要清理资料时无法操作。
2. **用户上传的文件无法预览**：AI总结资料有内容预览（Markdown渲染），但用户上传的文件（PDF、DOC、TXT等）只能下载，无法在页面内直接预览内容。
3. **课程卡片的"上传资料"按钮无实际功能**：点击后只显示alert提示"功能将在后续版本实现"，需要实现跳转到知识库并自动打开上传的功能。

## 变更范围

### 1. 资料删除功能
- 在用户上传资料的详情弹窗中添加"删除"按钮
- 删除后从IndexedDB中移除该资料条目
- 删除前弹出确认对话框防止误操作
- 删除后自动刷新资料列表

### 2. 文件预览功能
- 仿照AI总结文档的预览样式，为用户上传文件添加内容预览
- 支持的预览格式：
  - **TXT/MD**：直接在弹窗中渲染文本内容
  - **PDF**：使用 `<iframe>` 或 `<embed>` 嵌入PDF预览
  - **DOC/DOCX**：使用第三方在线预览服务（如 Google Docs Viewer 或 Microsoft Office Online）或降级为纯文本提取预览
- 预览区域与AI总结文档保持一致的样式（带边框、可滚动、最大高度限制）

### 3. 课程卡片上传跳转
- 修改课程卡片"上传资料"按钮的点击行为
- 点击后跳转到当前课程的知识库页面（`#/knowledge/:courseId`）
- 通过URL参数 `?upload=true` 触发自动打开上传区域
- 知识库页面加载时检测该参数，自动触发文件选择器

## 不涉及的范围

- AI生成资料的管理（不支持删除AI资料）
- 服务器端文件存储（仍使用IndexedDB本地存储）
- 文件格式转换功能
