## ADDED Requirements

### Requirement: 课程学习评估按钮显示
系统SHALL在每门课程学习页面的"查看资料"按钮右侧显示"课程学习评估"按钮。

#### Scenario: 评估按钮正确显示
- **WHEN** 用户进入课程学习页面
- **THEN** 在"查看资料"按钮右侧应显示"课程学习评估"按钮

#### Scenario: 评估按钮样式一致
- **WHEN** 评估按钮渲染完成
- **THEN** 按钮样式应与现有按钮风格一致，不影响页面布局

### Requirement: 学习数据收集
系统SHALL收集以下五个维度的学习数据：
1. 用户点击"去问AI"按钮的次数
2. 课程学习进度（三个阶段的完成情况）
3. 第三问题目答对情况和测验次数
4. 阅读知识库中的资料情况
5. 用户上传资料的情况

#### Scenario: AI咨询次数记录
- **WHEN** 用户点击"去问AI"按钮
- **THEN** 系统应记录该课程对应的AI咨询次数

#### Scenario: 学习进度记录
- **WHEN** 用户完成课程学习的某个阶段
- **THEN** 系统应记录该阶段的完成状态

#### Scenario: 测验成绩记录
- **WHEN** 用户完成第三问的测验题目
- **THEN** 系统应记录答对题目数量和总测验次数

#### Scenario: 资料阅读记录
- **WHEN** 用户阅读知识库中的资料
- **THEN** 系统应记录阅读的资料数量和类型

#### Scenario: 资料上传记录
- **WHEN** 用户上传学习资料
- **THEN** 系统应记录上传的资料数量和类型

### Requirement: AI评估报告生成
系统SHALL基于收集的学习数据，使用AI生成约500字的学习情况总结。

#### Scenario: 评估报告内容完整
- **WHEN** 用户点击"课程学习评估"按钮
- **THEN** 系统应生成包含以下内容的评估报告：
  - 学习优点
  - 学习不足
  - 需要提升的知识点
  - 学习建议

#### Scenario: 评估报告字数适当
- **WHEN** AI生成评估报告
- **THEN** 报告字数应在400-600字之间

#### Scenario: 评估数据准确
- **WHEN** AI生成评估报告
- **THEN** 报告应基于准确的学习数据，不包含虚构信息

### Requirement: 评估结果展示
系统SHALL在弹窗中展示AI生成的评估报告。

#### Scenario: 弹窗正确显示
- **WHEN** AI评估报告生成完成
- **THEN** 系统应弹出窗口显示评估报告

#### Scenario: 弹窗内容可读
- **WHEN** 评估报告显示在弹窗中
- **THEN** 内容应格式清晰，易于阅读

#### Scenario: 弹窗可关闭
- **WHEN** 用户查看完评估报告
- **THEN** 用户应能关闭弹窗返回学习页面
